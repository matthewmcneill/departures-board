---
description: Pick a specific session by ID and load its context and implementation plan into the current clean session for execution.
---

1. Identify the Target Session ID provided by the user in the command (e.g., `/queue-do-session [Target Session ID]`).
2. Execute the following core instruction:
   "Go and read ALL the content related to session [Target Session ID] for context and then bring the implementation plan here and review it."
   *(Note: Look in the brain folder, such as `.gemini/antigravity/brain/[SESSION_ID]/` to find the `implementation_plan.md`, `task.md`, and any other artifacts generated by that session)*
3. Read `.agents/queue.md`.
4. Check if the hardware lock in `.agents/queue.md` is held. If it is held by an old session that has seemingly finished, you may take over the lock.
5. Set the `Lock Status` in `.agents/queue.md`:
   - `Locked By`: [Current Session ID]
   - `Reason`: Executing implementation plan for [Target Session ID]
6. Find the target session in the `Pending Queue` and remove it from the list.
7. Notify the user: "Lock claimed. Context loaded. Starting implementation now."
8. Present the implementation plan to the user for review and approval.
