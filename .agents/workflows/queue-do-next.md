1. Read `.agents/queue.md`.
2. If there are items in the `Pending Queue`, get the Session ID of the first (top) item.
3. If the queue is empty, notify the user and stop.
4. Execute the following core instruction:
   "Go and read ALL the content related to session [SESSION_ID] for context and then bring the implementation plan here and review it."
   *(Note: Look in the brain folder, such as `.gemini/antigravity/brain/[SESSION_ID]/` to find the `implementation_plan.md`, `task.md`, and any other artifacts generated by that session)*
5. Check if the hardware lock in `.agents/queue.md` is held by another session. If it is held by an old session that has seemingly finished, you may take over the lock.
6. Set the `Lock Status` in `.agents/queue.md`:
   - `Locked By`: [Current Session ID]
   - `Reason`: Executing implementation plan for [Target Session ID]
7. Remove the target session from the `Pending Queue`.
8. Notify the user: "Lock claimed. Context loaded. Starting implementation now."
9. Begin executing the implementation plan.
